## 2. 核心算法详述

### 2.1 颜色匹配：从 RGB 到 CIEDE2000
传统的欧几里得距离在计算颜色差异时并不准确。为了实现“人眼看上去最接近”，必须将图像像素从 **RGB** 转为 **Lab** 色彩空间，并应用 **CIEDE2000** 公式计算色差 $\Delta E_{00}$。



公式简述：
$$\Delta E_{00} = \sqrt{\left(\frac{\Delta L'}{k_L S_L}\right)^2 + \left(\frac{\Delta C'}{k_C S_C}\right)^2 + \left(\frac{\Delta H'}{k_H S_H}\right)^2 + R_T \left(\frac{\Delta C'}{k_C S_C}\right) \left(\frac{\Delta H'}{k_H S_H}\right)}$$

* **实现逻辑**：遍历图片的每一个采样点（像素），将其与预设的拼豆色库（如 MARD 221 色）逐一比对，取 $\Delta E_{00}$ 最小值对应的色号。

### 2.2 图像采样与像素化
1.  **等比缩放**：使用 `canvas.drawImage` 将原图绘制到缩略画布上（尺寸由用户设定的“珠数”决定，如 $52 \times 52$）。
2.  **数据提取**：通过 `ctx.getImageData()` 获取原始 RGBA 数组。
3.  **网格映射**：将连续的像素信息离散化为二维矩阵，每个元素包含：`{ hex, colorCode, brandName, x, y }`。

## 3. 关键性能优化

### 3.1 Web Workers 并行计算
当处理高分辨率图纸（如 $100 \times 100$ 颗粒，即 10,000 个匹配计算）时，主线程会卡死。
* **方案**：将整个像素矩阵和色库传给 Worker。
* **优化**：使用 `Transferable Objects` 传递 `Uint8ClampedArray`，避免内存拷贝开销。

### 3.2 OffscreenCanvas 离屏渲染
在工作台进行缩放和拖拽预览时，直接操作 DOM 会导致帧率下降。
* **方案**：使用 `OffscreenCanvas` 在后台绘制带有网格、编号和豆子阴影的图层，再整体同步到主画布。

### 3.3 颜色查找表 (LUT) 缓存
* 如果用户反复切换同一张图的“亮度”或“对比度”，应对已计算过的 RGB -> 色号结果进行 Map 缓存，避免重复执行高能耗的 CIEDE2000 运算。

## 4. 模块功能设计

### 4.1 图像预处理 (Filter Module)
在像素化之前，提供类似 Photoshop 的滤镜功能：
* **亮度/对比度调节**：解决手机拍摄原图偏暗的问题。
* **色彩锐化**：增强轮廓，使生成的拼豆图纸边界更清晰。

### 4.2 导出模块 (Exporter)
* **SVG 矢量导出**：利用 Vue 的模板渲染能力，将像素矩阵转化为 `<svg>` 标签，确保打印时不失真。
* **PDF 生成**：使用 `jsPDF` 将统计清单（用量表）与图纸合并，生成可直接打印的 A4 预览件。

## 5. 开发里程碑 (Milestones)

1.  **Phase 1 (MVP)**：实现图片上传 + 基础像素化（RGB 距离匹配）+ Canvas 静态展示。
2.  **Phase 2 (Algorithm)**：集成 Lab 转换与 CIEDE2000 算法，加入主流品牌色库。
3.  **Phase 3 (Optimization)**：引入 Web Workers，增加工作台缩放/平移交互，支持“沉浸模式”。
4.  **Phase 4 (Final)**：导出功能完善，UI 细节打磨（适配移动端查看），部署至 Vercel。
